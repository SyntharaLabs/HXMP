# HXMP branding notes

Durable visual direction for HXMP marketing/token/project identity.

## Brand posture

Aim for IBM-like enterprise infrastructure: simple, clean, elegant, powerful, recognizable. The protocol can be soulful in copy, but the mark should feel institutional and trustworthy.

Avoid:

- crypto-bro visual language;
- mystical symbols as the primary mark;
- robot heads, brains, glowing eyes;
- complicated chain/cube clichés;
- generic AI-logo gradients;
- small illegible text inside token icons.

## Symbol direction

Strongest direction so far: a two-part identity rather than cramming all letters into one token icon.

- **Token/icon:** a simple geometric identity/memory mark, e.g. Keystone H or H/M ligature.
- **Wordmark:** separate readable `HXMP` text for headers, docs, and social graphics.

## Visual language

Use black/graphite/silver/white or restrained enterprise color. The user disliked obvious blue/cyan tech accents in generated logo drafts. If color is used, keep it subtle and premium, not glowy.

Good creative prompts should emphasize:

```text
old IBM / Bell Labs / ISO protocol standard / enterprise infrastructure badge
```

rather than:

```text
futuristic AI blockchain logo
```

## Copy line

Useful positioning:

```text
AgentID = identity anchor
HXMP = encrypted continuity layer
```

Marketing phrase:

```text
Persistent identity for autonomous agents.
```

or:

```text
Encrypted identity continuity for agents.
```
